## 1. 系統需求

| 項目 | 最低要求 | 建議 |
|:---|:---|:---|
| **Python** | 3.11+ | 3.12 |
| **Node.js** | 18+（僅 WhatsApp Bridge 需要） | 20 |
| **記憶體** | 256 MB | 1 GB |
| **CPU** | 1 核 | 1 核 |

## 2. 安裝方式

### 方式一：從原始碼安裝（推薦開發用）

```bash
git clone https://github.com/HKUDS/nanobot.git
cd nanobot
pip install -e .
```

### 方式二：使用 uv 安裝（快速穩定）

```bash
uv tool install nanobot-ai
```

### 方式三：從 PyPI 安裝

```bash
pip install nanobot-ai
```

### 安裝可選依賴

根據需要的 Channel 安裝對應依賴：

```bash
# WeChat (微信)
pip install nanobot-ai[weixin]

# 企業微信
pip install nanobot-ai[wecom]

# Matrix (Element)
pip install nanobot-ai[matrix]

# LangSmith 監控
pip install nanobot-ai[langsmith]
```

### 升級版本

```bash
# pip
pip install -U nanobot-ai

# uv
uv tool upgrade nanobot-ai

# 確認版本
nanobot --version
```

## 3. 初始化設定

### 自動初始化（推薦）

```bash
nanobot onboard
```

使用互動式引導精靈：

```bash
nanobot onboard --wizard
```

這會在 `~/.nanobot/` 建立以下結構：

```text
~/.nanobot/
├── config.json          # 主配置（API Keys, Model, Channels）
├── workspace/
│   ├── AGENTS.md        # Agent 行為規範
│   ├── SOUL.md          # Bot 人設與性格
│   ├── USER.md          # 用戶資訊
│   ├── TOOLS.md         # 工具使用指引
│   ├── memory/          # 長期記憶
│   ├── skills/          # 自訂技能
│   └── sessions/        # 對話歷史
├── history/             # 歷史日誌
├── media/               # 媒體文件
└── cron/                # 排程任務
```

### 手動初始化

如果不使用 `onboard`，手動建立配置：

```bash
mkdir -p ~/.nanobot/workspace/memory
```

然後建立 `~/.nanobot/config.json`（見下一節）。

| **OpenAI** | `openai` | [platform.openai.com](https://platform.openai.com) |

| **智譜 (Zhipu)** | `zhipu` | [open.bigmodel.cn](https://open.bigmodel.cn) |
| **通義千問 (DashScope)** | `dashscope` | [dashscope.aliyun.com](https://dashscope.aliyun.com) |
| **Moonshot / Kimi** | `moonshot` | [platform.moonshot.cn](https://platform.moonshot.cn) |
| **MiniMax** | `minimax` | [api.minimax.chat](https://api.minimax.chat) |
| **Mistral** | `mistral` | [console.mistral.ai](https://console.mistral.ai) |
| **Gemini** | `gemini` | [aistudio.google.com](https://aistudio.google.com) |
| **SiliconFlow** | `siliconflow` | [siliconflow.cn](https://siliconflow.cn) |
| **火山引擎** | `volcengine` | [volcengine.com](https://volcengine.com) |

### 配置範例

**OpenRouter（推薦全球用戶）：**

```json
{
  "agents": {
    "defaults": {
      "model": "anthropic/claude-opus-4-5",
      "provider": "openrouter"
    }
  },
  "providers": {
    "openrouter": {
      "apiKey": "sk-or-v1-xxx"
    }
  }
}
```

**騰訊混元（推薦中國大陸用戶）：**

```json
{
  "agents": {
    "defaults": {
      "model": "glm-5",
      "provider": "tencent_coding_plan"
    }
  },
  "providers": {
    "tencentCodingPlan": {
      "apiKey": "sk-xxx",
      "apiBase": "https://api.lkeap.cloud.tencent.com/coding/v3",
      "models": [
        {
          "id": "glm-5",
          "name": "GLM-5",
          "contextWindow": 202752,
          "maxTokens": 16384
        }
      ]
    }
  }
}
```

**Ollama（本地運行）：**

```json
{
  "agents": {
    "defaults": {
      "model": "llama3.1",
      "provider": "ollama"
    }
  },
  "providers": {
    "ollama": {
      "apiBase": "http://localhost:11434/v1"
    }
  }
}
```

## 5. Channel 設定

### Channel 總覽

| Channel | 配置鍵 | 認證方式 | 備註 |
|:---|:---|:---|:---|
| **Telegram** | `telegram` | Bot Token | 推薦，最穩定 |
| **WeChat (微信)** | `weixin` | QR Code 掃碼 | 需安裝 `[weixin]` 依賴 |
| **Discord** | `discord` | Bot Token | 需開啟 Message Content Intent |
| **WhatsApp** | `whatsapp` | QR Code 掃碼 | 需 Node.js 18+ |
| **Feishu (飛書)** | `feishu` | App ID + Secret | WebSocket 長連接 |
| **DingTalk (釘釘)** | `dingtalk` | App Key + Secret | — |
| **Slack** | `slack` | Bot Token + App Token | 支援 Socket Mode |
| **Matrix** | `matrix` | Access Token | 支援 E2EE |
| **Email** | `email` | IMAP/SMTP | 自動回覆 |
| **QQ** | `qq` | App ID + Secret | — |
| **企業微信** | `wecom` | Bot ID + Secret | 需安裝 `[wecom]` 依賴 |
| **Mochat** | `mochat` | Claw Token | 自動設定可用 |

### Telegram 設定（推薦）

1. Telegram 搜尋 `@BotFather` → `/newbot` → 複製 Token
2. 設定 `config.json`：

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "token": "YOUR_BOT_TOKEN",
      "allowFrom": ["YOUR_USER_ID"]
    }
  }
}
```

3. 啟動：`nanobot gateway`

### WeChat (微信) 設定

1. 安裝依賴：`pip install nanobot-ai[weixin]`
2. 設定 `config.json`：

```json
{
  "channels": {
    "weixin": {
      "enabled": true,
      "allowFrom": ["*"]
    }
  }
}
```

3. 啟動並掃碼：

```bash
nanobot gateway
# 首次啟動會顯示 QR Code，用微信掃碼登錄
```

> WeChat 顯示名稱由微信帳號暱稱決定，非 nanobot 配置項。修改：微信 App → 我 → 個人資料 → 名字

### 存取控制

所有 Channel 都支援 `allowFrom` 欄位：

```json
{
  "allowFrom": ["user_id_1", "user_id_2"]   // 僅允許特定用戶
  "allowFrom": ["*"]                          // 允許所有人
  "allowFrom": []                             // 拒絕所有人
}
```

## 6. 啟動與運行

### Gateway 模式（推薦 — 長期運行）

```bash
nanobot gateway
```

Gateway 會啟動所有已啟用的 Channel 並持續運行，包含：
- Channel 消息監聽
- Agent 消息處理循環
- Cron 排程任務
- Heartbeat 心跳

### CLI 互動模式

```bash
# 互動式對話
nanobot agent

# 發送單條消息
nanobot agent -m "今天天氣如何？"
```

### 查看狀態

```bash
nanobot status
```

## 7. Docker 部署

### 7.1 Docker Compose（推薦）

```bash
# 首次設定
docker compose run --rm nanobot-chat onboard
vim ~/.nanobot/config.json    # 加入 API Keys

# 啟動 Gateway
docker compose up -d nanobot-gateway

# CLI 工具
docker compose --profile cli run --rm nanobot-status
docker compose --profile cli run --rm nanobot-chat

# 查看日誌
docker compose logs -f nanobot-gateway

# 停止
docker compose down
```

`docker-compose.yml` 結構：

```yaml
x-common-config: &common-config
  build:
    context: .
    dockerfile: Dockerfile
  volumes:
    - ~/.nanobot:/root/.nanobot

services:
  nanobot-gateway:              # 長期運行服務
    container_name: nanobot-gateway
    <<: *common-config
    command: ["gateway"]
    restart: unless-stopped
    ports:
      - 18790:18790
    deploy:
      resources:
        limits:
          cpus: '1'
          memory: 1G

  nanobot-status:               # 一次性狀態查詢
    <<: *common-config
    profiles: [cli]
    command: ["status"]

  nanobot-chat:                 # 互動式對話
    <<: *common-config
    profiles: [cli]
    command: ["agent"]
    stdin_open: true
    tty: true
```

### 7.2 Docker（手動）

```bash
# 構建映像
docker build -t nanobot-ai:latest .

# 初始化（首次）
docker run -v ~/.nanobot:/root/.nanobot --rm nanobot-ai onboard
vim ~/.nanobot/config.json

# 啟動 Gateway
docker run -d \
  --name nanobot-gateway \
  -v ~/.nanobot:/root/.nanobot \
  -p 18790:18790 \
  --restart unless-stopped \
  nanobot-ai:latest gateway

# 發送單條消息
docker exec nanobot-gateway nanobot agent -m "Hello!"

# 查看狀態 / 日誌
docker exec nanobot-gateway nanobot status
docker logs -f nanobot-gateway
```

### 7.3 Dockerfile 分層快取

映像構建採用分層策略，修改源碼時只需重建最後兩層（約 30-60 秒）：

```text
Layer 1 — 基礎映像：uv:python3.12-bookworm-slim
Layer 2 — 系統依賴：Node.js 20, curl, git
Layer 3 — Python 依賴：uv pip install（快取層，依賴不變時跳過）
Layer 4 — 源碼安裝：nanobot/ + bridge/
Layer 5 — Bridge 構建：npm install + npm run build
```

映像大小約 800MB–1.2GB。

## 8. Apple Container 部署 (macOS 26+)

macOS 26+ 內建 Apple Container（`container` CLI），無需安裝 Docker Desktop。

### 8.1 構建映像

```bash
container build -t nanobot-ai:latest .

# 加速構建（分配更多資源）
container build -t nanobot-ai:latest -c 4 -m 4G .
```

### 8.2 建立容器

```bash
container create \
  --name nanobot-gateway \
  --cpus 1 \
  --memory 1G \
  --mount type=virtiofs,source=/Users/$USER/.nanobot,target=/root/.nanobot \
  --mount type=virtiofs,source=/Users/$USER/Desktop,target=/root/Desktop \
  --publish 18790:18790 \
  nanobot-ai:latest gateway
```

> **注意**：`--mount` 的 `source` 必須為**絕對路徑**，不支援 `~`

### 8.3 啟動與管理

```bash
# 啟動
container start nanobot-gateway

# 查看狀態
container exec nanobot-gateway /usr/local/bin/nanobot status

# 互動式對話
container exec -it nanobot-gateway /usr/local/bin/nanobot agent

# 發送單條消息
container exec nanobot-gateway /usr/local/bin/nanobot agent -m "Hello!"

# 查看日誌
container logs -f nanobot-gateway

# 資源監控
container stats nanobot-gateway

# 停止
container stop nanobot-gateway
```

### 8.4 Apple Container vs Docker 差異

| 特性 | Apple Container | Docker |
|:---|:---|:---|
| 掛載語法 | `--mount type=virtiofs,source=<path>,target=<path>` | `-v <host>:<container>` |
| 埠映射 | `--publish` 或 `-p` | `-p` |
| 容器內執行 | `container exec <id> <cmd>`（**不支援** `--`） | `docker exec <id> <cmd>` |
| 自動重啟 | 不支援（需用 launchd） | `--restart unless-stopped` |
| Compose | 不支援 | `docker compose` |
| 可執行路徑 | 必須用完整路徑 `/usr/local/bin/nanobot` | `nanobot` 即可 |

## 9. VPS 遷移部署

將本地環境遷移到 VPS 只需 3 樣東西：Docker 映像 + 配置目錄 + docker-compose.yml。

### 步驟

```bash
# 1. 本地構建並匯出映像
docker build -t nanobot-ai:latest .
docker save nanobot-ai:latest -o nanobot-ai.tar

# 2. 傳輸到 VPS
scp nanobot-ai.tar user@your-vps:/opt/nanobot/
scp docker-compose.yml user@your-vps:/opt/nanobot/
scp -r ~/.nanobot user@your-vps:~/.nanobot

# 3. VPS 上載入映像
ssh user@your-vps
docker load -i /opt/nanobot/nanobot-ai.tar

# 4. 設定配置權限
chmod 700 ~/.nanobot
chmod 600 ~/.nanobot/config.json

# 5. 啟動
cd /opt/nanobot
docker compose up -d nanobot-gateway

# 6. 驗證
docker exec nanobot-gateway nanobot status
docker logs -f nanobot-gateway
```

### 遷移檢查清單

- [ ] Docker 映像已構建並匯出
- [ ] `~/.nanobot` 配置目錄已複製
- [ ] API Keys 在目標環境可用（部分 Provider 有 IP 白名單）
- [ ] VPS 防火牆已開放 18790 埠（如需要）
- [ ] WeChat Channel 的 QR Code 需在新環境重新掃碼
- [ ] 容器日誌正常，無認證或網路錯誤

## 10. 服務自動啟動

### Linux — systemd

```bash
# 建立 service 檔案
mkdir -p ~/.config/systemd/user
cat > ~/.config/systemd/user/nanobot-gateway.service << 'EOF'
[Unit]
Description=Nanobot Gateway
After=network.target

[Service]
Type=simple
ExecStart=%h/.local/bin/nanobot gateway
Restart=always
RestartSec=10
NoNewPrivileges=yes
ProtectSystem=strict
ReadWritePaths=%h

[Install]
WantedBy=default.target
EOF

# 啟用並啟動
systemctl --user daemon-reload
systemctl --user enable --now nanobot-gateway

# 管理命令
systemctl --user status nanobot-gateway        # 查看狀態
systemctl --user restart nanobot-gateway       # 重啟
journalctl --user -u nanobot-gateway -f        # 查看日誌

# 保持登出後運行
loginctl enable-linger $USER
```

### macOS — launchd

```bash
# 建立 plist 檔案
cat > ~/Library/LaunchAgents/com.nanobot.gateway.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.nanobot.gateway</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/container</string>
        <string>start</string>
        <string>nanobot-gateway</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <false/>
</dict>
</plist>
EOF

# 啟用
launchctl load ~/Library/LaunchAgents/com.nanobot.gateway.plist

# 管理命令
launchctl list | grep nanobot                  # 查看狀態
launchctl unload ~/Library/LaunchAgents/com.nanobot.gateway.plist  # 停用
```

| launchctl 命令 | 作用 | 對應 systemctl |
|:---|:---|:---|
| `launchctl load <plist>` | 註冊並啟用 | `systemctl enable --now` |
| `launchctl unload <plist>` | 停用並移除 | `systemctl disable --now` |
| `launchctl list \| grep nanobot` | 查看狀態 | `systemctl status` |

## 11. 本地目錄掛載

容器預設掛載 `~/.nanobot` 作為配置目錄。如需讓容器存取其他目錄（如 Desktop、Documents），需在建立容器時指定。

> 容器不支援動態新增掛載，需**刪除舊容器後重建**。映像和配置不受影響。

### Apple Container

```bash
container stop nanobot-gateway
container rm nanobot-gateway
container create --name nanobot-gateway \
  --mount type=virtiofs,source=/Users/$USER/.nanobot,target=/root/.nanobot \
  --mount type=virtiofs,source=/Users/$USER/Desktop,target=/root/Desktop \
  --mount type=virtiofs,source=/Users/$USER/Documents,target=/root/Documents \
  --publish 18790:18790 \
  nanobot-ai:latest gateway
container start nanobot-gateway
```

### Docker

```bash
docker run -d --name nanobot-gateway \
  -v ~/.nanobot:/root/.nanobot \
  -v ~/Desktop:/root/Desktop \
  -v ~/Documents:/root/Documents \
  -p 18790:18790 \
  --restart unless-stopped \
  nanobot-ai:latest gateway
```

### Docker Compose

在 `docker-compose.yml` 的 `volumes` 加入：

```yaml
volumes:
  - ~/.nanobot:/root/.nanobot
  - ~/Desktop:/root/Desktop
  - ~/Documents:/root/Documents
```

## 12. 自訂與個性化

### 配置變更等級

| 變更項目 | 檔案位置 | 重啟容器？ | 重建映像？ |
|:---|:---|:---|:---|
| Bot 人設 / 性格 | `~/.nanobot/workspace/SOUL.md` | 否 | 否 |
| Agent 行為規範 | `~/.nanobot/workspace/AGENTS.md` | 否 | 否 |
| 用戶資訊 | `~/.nanobot/workspace/USER.md` | 否 | 否 |
| 自訂技能 | `~/.nanobot/workspace/skills/` | 否 | 否 |
| API Keys / Model | `~/.nanobot/config.json` | **是** | 否 |
| Channel 啟用/停用 | `~/.nanobot/config.json` | **是** | 否 |
| Logo Icon | `nanobot/__init__.py` → `__logo__` | — | **是** |
| Python 源碼 | `nanobot/**/*.py` | — | **是** |

### 修改 Bot 人設

編輯 `~/.nanobot/workspace/SOUL.md`（即時生效，無需重啟）：

```markdown
# Soul

I am 小助手, a personal AI assistant created by Dany.

## Personality
- Helpful and friendly
- Reply in Traditional Chinese by default
- Concise and to the point

## Values
- Accuracy over speed
- User privacy and safety
```

### 修改 Logo Icon

編輯 `nanobot/__init__.py`：

```python
__logo__ = "🤖"  # 或 🧠 ⚡ 🦊 🐱
```

修改後需重建映像並重建容器。

## 14. 常見問題排查

### 容器內 `nanobot` 命令找不到

```text
Error: HTTP request to https://registry-1.docker.io/v2/library/nanobot/manifests/latest
```

**原因**：`nanobot` 被當作映像名而非命令
**解決**：使用完整路徑 `/usr/local/bin/nanobot`

### `container exec` 加 `--` 報錯

```text
failed to find target executable --
```

**原因**：Apple Container 的 `exec` 不支援 `--` 分隔符
**解決**：直接 `container exec <id> <cmd>`，不加 `--`

### WeChat 掃碼後連線中斷

**可能原因**：
- Token 過期 → 重新掃碼登錄
- 網路不穩 → 檢查容器 DNS 設定
- `errcode == -14` → 暫停 1 小時後自動恢復

### 修改 config.json 後沒有生效

**原因**：`config.json` 在 Gateway 啟動時載入一次
**解決**：重啟容器

```bash
# Apple Container
container stop nanobot-gateway && container start nanobot-gateway

# Docker Compose
docker compose restart nanobot-gateway
```

### 容器無法存取本地檔案

**原因**：目錄未掛載到容器
**解決**：見 [11. 本地目錄掛載](#11-本地目錄掛載)

### Docker 構建失敗

```bash
# 清除快取重建
docker build --no-cache -t nanobot-ai:latest .

# Apple Container
container build --no-cache -t nanobot-ai:latest .
```
